src.data.gcp_strava_data_load_preprocess.generate_employment_prediction_model_data(activity_df, start_year, end_year, start_month, end_month, label)¶

Used to generate month by month data of activities per hour of the day, both in terms of percentage of activites started at each hour/day of the week, as well as raw counts of activities per hour of the day for training the employment prediction models.

Parameters

  • activity_df (DataFrame) – The pre-processed Strava activity data frame.

  • start_year (int) – Year in which to start processing data.

  • end_year (int) – YEar in which to stop processing data from Strava data.

  • start_month (int) – Month for which to start processing Strava data.

  • end_month (int) – Month for which to end processing of strava data in end_year

  • label (int) – 0 for unemployed, 1 for employed

Returns

indexed by year, month, day of week, hour and how many activities were started then train_data (list): activities by hour of day 0-23, then label, year, month.

Return type

summary_data (dict)

src.data.gcp_strava_data_load_preprocess.preprocess_employment_model_data(input_data, work_hours)¶

Breaks out hourly percentage of activities started and uses the currently set work hours to return breakdown of morning vs. afternoon activities.

Parameters

  • input_data (DataFrame) – Raw data with 0-23 columns for each hour of the day, label, year and month columns.

  • work_hours (List) – A list of two windows for morning/aft work hours, e.g. [[9, 11], [13, 16]]

Returns

Tuple of the processed dataframe with new columns, and the labels corresponding to the data.

Return type

DataFrame, Series

src.data.gcp_strava_data_load_preprocess.preprocess_strava_df(raw_df, min_act_length=1200, max_act_dist=400, export=False)¶

Processes the raw BigQuery data and manages feature additions, scaling etc. for use in the dashboard.

Parameters

  • raw_df (DataFrame) – The extracted data from BigQuery

  • min_act_length (int, optional) – THe shortest activity to include in the results in seconds. Defaults to 1200 (20min).

  • max_act_dist (int, optional) – Longest activity allowed to catch data issues/outliers, in km . Defaults to 400.

  • export (bool, optional) – Used to export the processed df to CSV for local work/offline prototyping. Defaults to False.

Returns

The cleaned and processed data with additional features etc. added.

Return type

DataFrame
