src.pages.employment_prediction module¶

src.pages.employment_prediction.update_employment_page(_page_load, strava_dict, empl_dict)¶

Generates the pages on very first load from the unused page-load-div. Also initializes persistent cached data if it’s not already loaded by another page.

Parameters

  • _page_load (None) – Placeholder to trigger generation of plots on page load with loading symbols

  • strava_dict (dict) – The cached strava data if already loaded, can be None if not loaded by another page already.

  • empl_dict (dict) – The static employment data that can be used by multiple pages for faster loading of pages.

Returns

multiple primary figures are returned for the employment prediction page.

Return type

Plotly Figures

src.pages.homepage module¶

src.pages.homepage.initial_page_load(_page_load, strava_dict, empl_dict, kudos_dict)¶

Loads initial data for supporting visuals and caches re-used data. Updates the blank placeholder plots so loading symbols work as expected and the page can load before data is loaded.

Parameters

  • _page_load (None) – placeholder input for on initial page-load the callback is triggered

  • strava_dict (dict) – Strava data stored in the dcc.Store component for reuse.

  • empl_dict (dict) – employment data for employment info and for re-use on other pages.

  • kudos_dict (dict) – Actual vs predicted kudos data to be shared between pages.

Returns

outputs the primary data stores for re-use in dict form. Figures: multiple output figures for display on initial page load.

Return type

data

src.pages.homepage.strava_eda_update(y_label, year_range, group_by, strava_dict)¶

Updates the EDA plot of Strava data on interaction, also if the cached Strava data is updated, like on initial page load this should trigger and update the blank placeholder plot right away.

Parameters

  • y_label (string) – the data field from strava data to plot on y-axis

  • year_range (list) – start/end year of data to be plotted.

  • group_by (str) – what time period (week, day, month) to group activity info by

  • strava_dict (dict) – the cached strava data

Returns

the Strava activity eda figure.

Return type

figure

src.pages.kudos_prediction module¶

src.pages.kudos_prediction.update_predicted_kudos_number(num_followers, custom_name_bool, distance, elevation, achievements, _children, kudos_static_dict)¶

Accepts changed features for predicting kudos and shows updated predicted kudos numbers along with the optimization plots where max kudos can be gained.

Parameters

  • num_followers (int) – How many followers the user entered.

  • custom_name_bool (int) – 0 for standard name, 1 for custom name

  • distance (int) – distance in kilometers

  • elevation (int) – elevation gain in meters

  • achievements (int) – no. of achievements gained on the activity

  • _children (None) – Placeholder for a div to trigger on page load instead of interaction.

  • kudos_static_dict (dict) – the cached static kudos prediction data for fast reloading.

Returns

multiple figures are returned with updated data etc. dict (dcc.Store): the updated cached data is returned so further interactions don’t trigger a reload of the data.

Return type

figures

src.pages.layouts module¶

src.pages.layouts.blank_placeholder_plot(background_color='white')¶

Creates an empty plot to be used on page load before callbacks complete, allows page load and all figure updates/data loading to be managed in callbacks without empty plots being shown.

Parameters

background_color (str, optional) – What color the background should match to be “invisible” until updated. Defaults to “white”.

Returns

Empty plot with lines removed and formatting updated to be blank.
